## Описание

Реализация простой модульной архитектуры.

### Модули

Модуль - это отдельный роут или часть переиспользуемого функционала со своей бизнес-логикой. Если функционал связан одним базовым роутом (отображается в дочернем роуте, например, auth, auth/login, auth/restore), то дочерние роуты выносятся в отдельные модули.

Модуль экспортирует свой компонент (можно обернуть в lazy для вынесения в отдельный chunk) и reducer. Reducer модуля регистрируется в глобальном хранилище приложения динамически при старте. Бизнес-логика модуля реализуется в его slice.

### Страницы

Страница - это компоновка модуля/модулей в единое представление.

Страница экспортирует описание роутинга и импортирует модули. Также реализует отображение компонентов дочерних роутов через Outlet. (?)
